# Branch: `original-design-mirror`

이 브랜치는 `main` 의 새 디자인을 그대로 두지 않고, **원본 mercury.or.kr 사이트와
가능한 한 유사한 외형**으로 다듬는 작업 전용 브랜치이다.

`main` 브랜치는 건드리지 않으며, 본 브랜치의 결과물은 Pull Request 로만 검토하고
직접 머지하지 않는다.

---

## 작업 범위 — 권리 문제 없이 진행 가능한 항목

다음 항목은 외부 자산을 복제하지 않고도 충실하게 맞출 수 있는 영역이며, 본
브랜치에서 우선 진행한다.

1. **URL · 메뉴 구조 정합성**
   - 메인/소개/수은이란/콘텐츠/게시판의 라우트 구성과 라벨이 원본과 1:1 매칭되는지
     재점검하고 누락된 서브 메뉴(있다면)를 추가.
   - 그누보드 스타일 PHP 쿼리스트링 → 새 라우트 매핑(`next.config.mjs`) 보강.

2. **레이아웃 골격**
   - 헤더 / 글로벌 네비 / 좌측 서브 네비 / 본문 / 푸터 구획 비율 점검.
   - 게시판 테이블 컬럼 구성·정렬·공지 스타일을 일반적인 한국 공공·보건 정보
     사이트 컨벤션에 맞춰 다듬기.
   - 페이지 헤더(브레드크럼 + 타이틀 + 서브타이틀) 영역 통일.

3. **타이포그래피 / 색상 톤(원본 자산 미포함)**
   - 본문 폰트 패밀리(Pretendard, Noto Sans KR 등 OFL/오픈 폰트) 우선 사용.
   - 로고·이미지 등 보호 대상 자산을 도용하지 않는 범위에서, 한국 공공/보건
     정보 사이트의 일반적 청록·블루 톤을 가다듬는다.
   - 헤더·푸터 높이, 메뉴 hover/active 인터랙션을 깔끔하게 정리.

4. **반응형 / 접근성 / 인쇄 보강**
   - 모바일 메뉴 토글, 작은 화면에서의 게시판 테이블 동작 점검.
   - 콘트라스트, 키보드 포커스, 스크린리더용 라벨 추가.

5. **PHP 호환 라우트 추가**
   - 만약 원본에서 `/m/index.php` (모바일), `/sub/...` 같은 경로가 더 있다면
     redirect 추가.

## 권리 확인이 필요한 항목 — 사용자 결정 후 진행

다음 항목은 **외부 사이트의 보호 자산**(이미지·로고·일러스트·고유 텍스트)을
포함할 가능성이 있어, 본 브랜치 작업자(또는 사이트 운영 권한자)가 직접 자료를
공급해야 한다. 자료가 제공되기 전까지는 자리(slot)만 잡아두고 콘텐츠는
플레이스홀더로 둔다.

| 항목 | 진행 조건 |
|---|---|
| 헤더 로고 이미지 | 운영자가 로고 PNG/SVG 파일 제공 시 `public/logo.svg` 로 추가 후 `SiteHeader.tsx` 교체 |
| 메인 히어로 사진/일러스트 | 사용 권한 있는 이미지(자체 제작 또는 라이선스 보유 자산)만 추가 |
| 메인 컬러 정확한 헥사 | 운영자가 보유한 브랜드 가이드(있다면) 제공 시 변수만 교체 |
| 인사말 본문, 각 정보 페이지 본문 | 운영자가 보유한 원고를 붙여 넣기 (관리자 글쓰기 또는 직접 페이지 수정) |
| 게시판 기존 글, 첨부파일 | 별도 마이그레이션 절차(`scripts/import-legacy-db.js`) 사용 |

## 진행 원칙

- 본 브랜치는 **`main` 으로 직접 머지하지 않는다.** PR 단계까지만 준비한다.
- Vercel 은 PR/브랜치 단위로 **Preview Deployment** 를 자동 생성한다. 푸시 후
  Vercel 대시보드 또는 PR 페이지의 "Vercel" 봇 코멘트에서 미리보기 URL 을 확인할
  수 있다.
- 자산 도용 위험이 있는 작업은, 운영자가 자료를 직접 공급한 뒤에만 시작한다.
